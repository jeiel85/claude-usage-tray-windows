> **#27**: 코드 확인 결과 이미 `static readonly HttpClient`로 수정되어 있어 이슈만 닫음.
>
> **#39 추가 사용량**: API 응답 확인 결과 `used_credits: 699, monthly_limit: null, utilization: null`. 한도 미설정 시 프로그레스 바/퍼센트를 숨기고 "X 크레딧 사용됨 (한도 미설정)" 텍스트만 표시하도록 수정. `ExtraHasLimit` 프로퍼티 추가.
>
> **계정 전환 버그**: 두 가지 원인 수정.
> 1. `CredentialService` — `Renamed` 이벤트 누락. Claude 앱(Electron)은 임시 파일에 쓴 뒤 rename하는 atomic write 방식을 사용하므로 `Changed/Created` 이벤트가 발생하지 않음. `Renamed` 이벤트 핸들러 추가.
> 2. `MainViewModel.RefreshAsync` — 이전 계정에서 발생한 `Retry-After` 대기가 새 계정 API 호출을 막는 버그. 계정 전환 시 `_apiRetryAfter` 초기화.
> 3. 폴백 추가 — FileSystemWatcher가 이벤트를 놓쳐도 2분 정기 새로고침 시 `orgUuid` 비교로 계정 전환 자동 감지.
>
> **계정 이름 기능 제거**: 실용성 낮아 Settings UI, `RenameCurrentAccount`, `AccountNames`, 관련 로컬라이제이션 전부 삭제.

### 🔍 미해결 — 5h/7d 표시값이 Claude 공식 UI와 다름 (2026-04-02 현재 조사 중)

**증상**: 앱의 5시간(0%), 7일(100%) 수치가 claude.ai 설정 → 사용량 화면(5h 75%, 7d 96%)과 다름.

**확인된 사실**:
- 계정 전환 자체는 정상 작동 (추가 사용량이 계정 1: 699크레딧 → 계정 2: 0크레딧으로 변경됨)
- API 응답 `five_hour.resets_at: null`, `utilization: 0.0` — 버킷이 리셋되었거나 아직 집계 전 상태로 추정
- Claude 공식 UI의 "현재 세션(롤링 윈도우)"과 API의 `five_hour`(고정 5시간 버킷)이 서로 다른 데이터를 보여줄 가능성 있음

**다음 확인 필요**:
- 계정 1로 다시 전환 시 앱 5h 값이 바뀌는지 → 바뀌면 계정 전환 코드는 정상, API 설계 차이가 원인
- API 엔드포인트(`/api/oauth/usage`)의 `five_hour` 필드가 Claude UI의 "현재 세션"과 동일한 윈도우인지 추가 검증 필요
